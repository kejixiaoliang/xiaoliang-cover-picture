# 科技小亮封面配图

这个技能用来在 Codex 里直接生成最终可用的封面图片，而不是只写提示词。默认使用内置 `image_gen` 工具；除非用户明确要求命令行或 API 方案，否则不要要求用户提供 `OPENAI_API_KEY`。

核心审美：干净、易读、有传播钩子、以创作者为主角。公众号封面和小红书封面都不能做成普通办公桌测评照；要优先选择一个大胆但清晰的视觉隐喻，让科技小亮本人、标题和主题物件形成一眼能读懂的封面。

## 默认参考体系

- 默认 IP 参考图：`assets/ip-reference/xiaoliang-ip-reference.png`
- 默认正向示例目录：`examples/approved/`
- 默认反例示例目录：`examples/rejected/`
- 这些素材是本地私有工作文件，不属于公开仓库内容。
- 如果 `assets/ip-reference/xiaoliang-ip-reference.png` 不存在，在生成需要稳定人物身份的封面前，先提醒用户把清晰参考图放到这个路径。
- 如果用户提供了另一个本地参考图路径，本次任务优先使用用户提供的路径，并说明默认复用路径仍然是 `assets/ip-reference/xiaoliang-ip-reference.png`。
- 每次生成前，都要默认查看 `examples/approved/`，从已认可完整封面中提取审美规律、标题尺度、人物稳定性、构图张力和平台差异；不要等用户要求才看。
- 每次生成前，都要默认查看 `examples/rejected/`，从反例中提取需要避开的失败模式；不要模仿反例画面。
- 如果 `examples/approved/` 或 `examples/rejected/` 里暂时没有图片，继续执行；主要依赖 `references/cover-style.md`、`references/prompt-patterns.md`、`references/style-matrix.md` 和 `references/quality-gate.md`。
- 把参考图当作严格身份约束：圆脸、短而略蓬松的黑发、银框眼镜、年轻科技创作者气质、自然皮肤质感、干净黑色上衣。
- 使用本地参考图前，先用 `view_image` 检查，让它进入当前对话的视觉上下文。
- 使用正向或反例样例前，也要先用 `view_image` 检查被选中的样例图；正向样例用于提取规律，反例样例用于提取避雷点。

如果用户在对话里上传了另一张参考图，把用户上传的图作为主要身份参考，本地默认图只作为兜底。

## 核心流程

1. 读取用户给出的文章标题、主题、摘要、Markdown 或草稿。
2. 判断文章类型：AI 教程、产品测评、工具体验、观点分析、趋势解读、失败复盘、硬件制作、创作者发布。
3. 默认读取参考体系：
   - 检查 `assets/ip-reference/xiaoliang-ip-reference.png`，确保人物身份有依据；
   - 查看 `examples/approved/`，总结本次应该继承的正向规律，例如大标题尺度、人物距离、光线、单一物件、平台构图；
   - 查看 `examples/rejected/`，总结本次必须避开的旧问题，例如人物不像、标题小、画面乱、手机滥用、构图平淡；
   - 只提取规律和避雷点，不照抄任何一张样例的构图。
4. 先产出一个简短封面概念，至少包含：
   - 最终封面大标题，拆成 2-4 行中文短句；
   - 一个需要高亮的关键词或短语；
   - 科技小亮的表情、身体姿态和手部动作；
   - 一个符合文章主题的真实或超现实真实场景，不能习惯性复用同一套画面；
   - 只保留一个核心视觉主体；
   - 一个能把文章洞察变成画面的大胆隐喻。
5. 除非用户只要一种格式，否则同时生成：
   - 公众号封面：`2.35:1`，横向宽画幅；
   - 小红书封面：`3:4`，竖向移动端画幅。
6. 每种格式分别调用一次内置 `image_gen`。两种比例必须分别写提示词，因为构图不能只是裁切。
7. 检查生成图：人物身份、标题可读性、构图干净度、视觉钩子、是否继承正向样例规律、是否避开反例问题。
8. 如果图像杂乱、文字太多、过于普通、像广告、人物不像参考图、姿势重复、没有主题张力，或重复了 `examples/rejected/` 中的旧问题，针对失败点重试一次。
9. 对项目内产物，把最终图片移动或复制到 `outputs/`，使用有意义的文件名，并向用户报告保存路径。

## 提示词要求

写图像提示词前先阅读 `references/cover-style.md`。选择场景、动作和标题结构时阅读 `references/prompt-patterns.md`。选择文章类型、平台布局、姿势变化和标题模式时阅读 `references/style-matrix.md`。接受或交付封面前使用 `references/quality-gate.md` 检查。

每个提示词必须包含以下信息：

```text
用途：营销配图
资产类型：中文创作者文章封面
参考图：使用科技小亮 IP 参考图作为严格身份约束
画幅比例：<2.35:1 或 3:4>
主体人物：科技小亮，轻微美化但仍然可识别的年轻中文科技创作者
画面文字（逐字）："<最终标题，保留换行>"
构图：大标题 + 科技小亮 + 一个核心视觉主体 + 极简环境
风格：干净真实的编辑部封面；当主题有发现、转化、记忆、产品洞察或“AI 看透某事”的能量时，使用高冲击的真实或超现实真实画面
身份约束：保留圆脸、短而略蓬松的黑发、银框眼镜、黑色上衣、年轻且接地气的科技创作者气质
避免：漂浮界面卡片、代码框、图标拼贴、复杂背景、小字、廉价广告感、普通商务肖像、网红脸
```

## 身份与美化规则

- 可以轻微美化：皮肤更干净、眼睛更有神、发型更整洁、姿态更好、光线更专业。
- 必须保留身份：圆脸轮廓、银色圆框眼镜、短黑发、年轻年龄感、自然表情、接地气的科技创作者气质。
- 不要把人物变成时尚模特、动漫角色、公司宣传照男主、年长商务人士或泛化网红。
- 表情自然：专注、好奇、自信、轻微兴奋、发现问题时的惊讶、分析感。
- 姿势要随文章主题变化，不要反复使用僵硬点赞、夸张指屏幕、无理由拿手机。
- 如果用户说人物不像小亮，重试时强化身份描述：“更接近参考照片，而不是泛化帅气网红；保留更圆的脸、自然皮肤质感、短而略蓬松的头发、银框眼镜、黑色拉链外套。”

## 姿势和场景变化

默认不要复用同一个姿势公式。写提示词前，从每组里选一个具体选项，并和文章主题绑定：

- 身体姿态：坐姿近景、站立半身、工作台侧脸、俯拍手部加局部脸、越肩视角、低角度主角视角、蹲在硬件旁边、后仰意识到问题、从桌面转身看向镜头。
- 手部动作：拿起核心物件、插入线缆、焊接、用探针测量、打开盖子/书/盒子、拉出线缆、放置芯片、比较两个物件、画出连接线、举起完成品、用手遮挡强光、按下一个实体按钮。
- 表情：发现感、“等等这居然能用？”、克制惊讶、完成后的自豪、专注调试、怀疑式测试、被巧思逗笑、冷静讲解。
- 镜头角度：低角度前景物件、桌面俯拍、三分之四侧面、脸部近景加物件、宽阔创客房间、硬件微距前景且小亮在后方、镜面/反射画面、门口或工作室揭示。

场景库：

- 硬件/创客：焊接垫、示波器风格工作台、USB 线和 ESP32、工具挂板、被物件照亮的深色桌面、带零件盘的日光桌面、双手组装电路板。
- AI/工具：干净工作室桌面、发光纸质笔记本、实体工作流地图、档案盒、放大镜/显微镜观察工具、单个设备上的聚光。
- 阅读/知识：打开的书、书架阴影、台灯、批注卡片、纸张堆、像图书馆的一角。
- 产品/测评：桌面产品舞台、拆解托盘、对比尺、打开的包装盒、小型柔光箱。
- 观点/趋势：干净墙面、站立讲解姿态、大型象征物、窗边光、简洁工作室角落。

同一轮任务要连续输出多张图时，至少主动变化两项：身体姿态、手部动作、镜头角度、场景类型、光源。

## 构图规则

- 第一眼必须干净。
- 一张封面只表达一个主视觉想法。
- 优先使用真实环境：桌面、笔记本电脑、手机、桌上产品、工作室角落、家庭办公室、简单测评布景。
- 除非用户明确要求，否则避免漂浮窗口、界面卡片、代码片段、仪表盘、图表面板和图标群。
- 中文大标题是主视觉之一，要短、粗、清晰。
- 标题优先白色，最多配一个强调色：亮蓝、青色、橙红。
- 除非用户明确要求，不要在图里加入副标题或小字说明。
- 工具类文章不要默认“创作者拿手机”。只有手机本身是被测产品，或用户明确要求时才使用。
- 如果文章有强烈的转化、发现、记忆或“AI 看透某事”的角度，用实体隐喻表达：打开的书、入口、显微镜、聚光灯、爆炸拆解物件、实验桌、镜子、档案盒、发光地图。

## 冲击力默认值

两种格式都默认先追求冲击力。干净不等于平淡。

通用默认值：

- 用一个隐喻或实体物件让读者一眼理解文章。
- 让小亮用具体动作和物件互动，不要每次都只是张开双手。
- 对比度要比普通桌面照片更强，但画面仍然清楚可信。
- 避免被动坐姿、僵硬指点、无理由手持手机。

公众号封面：

- 利用宽画幅做左右张力：一侧巨大标题，另一侧是小亮和核心物件；也可以让斜向物件穿过画面。
- 标题要能承受公众号裁切，建议 2-3 行大字，不要很多小字。
- 核心物件放在前景或中左/中右，让画面有纵深，不要是平铺肖像。
- 优先选择明确的故事瞬间：打开、揭示、点亮、拆解、接线、焊接、测量、比较、举起结果、发现问题。
- 除非文章是纯观点评论，否则不要接受平静办公室墙面肖像。

小红书封面：

- 用近景或中近景，让脸部可识别。
- 标题放在上方 40%-50%，使用超大中文粗体。
- 加一个有斜线或前景纵深的核心物件。
- 对比度比公众号更强，但不能脏乱。
- 小亮必须和核心物件有明确互动，并且动作要符合文章。不要每张都“前倾张开双手”。

微信读书技能封面的成功模式：

- 如果存在，可参考 `examples/approved/weread-skill-xiaohongshu-impact-clean.png` 的审美方向。
- 主题：AI 读取个人阅读历史。
- 标题：`微信读书技能\n把我的书架\n看透了`
- 视觉隐喻：一本巨大的打开书本发出青蓝光，像里面装着用户阅读数据。
- 动作：小亮朝书本前倾，双手打开，惊讶但专注。
- 公众号改编：同一个隐喻改成宽画幅，左侧巨大标题，右侧/中部是小亮和发光打开的书，强斜向光，不出现手机。
- 有效原因：标题强、隐喻单一、无手机、脸部可识别、对比强、缩略图冲击力清楚。

## 格式指南

公众号 `2.35:1`：

- 使用有冲击力的宽构图，不要做成稳定普通桌面肖像。
- 大标题放一侧或左/中区域，必须巨大且可读。
- 小亮和核心视觉主体放在另一侧或中左/中右，并有明确动作。
- 用前景纵深、斜向光或单个大胆物件制造张力。
- 背景保持稀疏，冲击力来自标题、小亮和一个隐喻物件。

小红书 `3:4`：

- 比公众号更紧、更有力量，适合移动端近景。
- 标题占上方 40%-50%，最强短语要最大。
- 小亮放在中部/下部，有清楚、有能量的动作。
- 核心视觉主体在前景足够大。
- 默认用创意真实或超现实真实场景，不要普通桌面截图。

## 质量门槛

以 `references/quality-gate.md` 作为最终验收清单。

出现以下情况要拒绝并重试：

- 拼贴杂乱或物件太多；
- 漂浮卡片、界面面板、代码框、图标堆；
- 标题太小、太密或不可读；
- 人物不像参考图；
- 小亮变成泛化网红、商务图库男或年长商务人士；
- 无强理由只是拿着手机；
- 姿势和上一张成功封面过于接近；
- 只会前倾张开双手，而文章明明适合别的动作；
- 无主题理由地重复深色桌面/发光物件套路；
- 封面太稳、太普通、太像桌面测评照；
- 过度发光、赛博朋克、电影海报感、廉价促销风；
- 肢体僵硬或手势像图库照片；
- 背景和标题或人物抢注意力。

重试时只改失败点。文章概念和标题尽量保持稳定。
