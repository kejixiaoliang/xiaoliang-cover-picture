# 封面质量门槛

接受生成结果前，必须用这份清单检查。关键项失败时，不要交付，直接针对失败点重试。

## 1. 生成前检查

| 检查项 | 通过标准 |
| --- | --- |
| IP 参考图存在 | `assets/ip-reference/xiaoliang-ip-reference.png` 存在，或用户提供了其他参考图路径。 |
| 私有素材保持本地 | 参考图、生成封面、文章截图和草稿没有被准备上传到公开仓库。 |
| 输出路径是本地路径 | 最终封面保存在 `outputs/`，或用户明确指定的其他本地路径。 |
| 示例库路径清楚 | 已认可完整封面放在 `examples/approved/`；反例封面放在 `examples/rejected/`。 |

如果默认 IP 参考图不存在，在生成前暂停，并提醒用户放置清晰参考图：

```text
assets/ip-reference/xiaoliang-ip-reference.png
```

## 2. 提示词检查

| 检查项 | 通过标准 |
| --- | --- |
| 文章类型明确 | 已判断为 AI 教程、产品测评、工具体验、观点分析、趋势解读、失败复盘、硬件制作或创作者发布。 |
| 标题足够短 | 2-4 行中文短句；除非用户明确要求，不加入长副标题或小字说明。 |
| 只有一个核心主体 | 提示词里只指定一个主要物件、产品、隐喻或场景锚点。 |
| 动作清楚 | 小亮正在打开、测试、连接、比较、揭示、测量、讲解或回应核心主体。 |
| 平台比例明确 | 公众号提示词写明 `2.35:1`；小红书提示词写明 `3:4`。 |
| 避免项完整 | 明确禁止漂浮界面卡片、代码框、图标拼贴、复杂背景、小字、泛化网红脸和无理由拿手机。 |

## 3. 图片验收

| 区域 | 需要拒绝的情况 |
| --- | --- |
| 标题 | 文字太小、太密、错字、拥挤，或缩略图不可读。 |
| 人物 | 看起来像泛化网红、年长商务人士、时尚模特、动漫角色或无关图库肖像。 |
| 构图 | 没有视觉焦点，道具太多，界面元素太多，或只是平铺的左字右人。 |
| 场景 | 没有主题理由地重复“深色桌面 + 蓝色发光 + 张开双手”。 |
| 平台适配 | 公众号缺少宽画幅张力，或小红书不够近、不够移动端。 |
| 隐私 | 图片里出现了无意的私人截图、个人信息、文件路径或用户数据。 |

## 4. 重试规则

只重试失败点。除非标题本身失败，否则文章概念和标题保持稳定。

| 失败点 | 重试指令 |
| --- | --- |
| 人物不像参考图 | 强化身份约束：更接近参考照片，保留更圆的脸、自然皮肤质感、短而略蓬松的黑发、银框眼镜、接地气科技创作者气质。 |
| 标题不可读 | 缩短标题，放大字号，删掉副标题，只保留一个高亮短语。 |
| 画面杂乱 | 删除多余屏幕、图标、面板、道具和二级说明，只保留一个核心物件。 |
| 构图太被动 | 增加一个实体动作：打开、插入、测量、拆解、按下、比较、举起或揭示。 |
| 平台不匹配 | 重新构图而不是裁切。公众号需要宽画幅张力；小红书需要更近的标题、人物和物件关系。 |

## 5. 最终交付

只报告有用的最终路径和必要说明：

```text
公众号封面：outputs/<name>-wechat-cover.png
小红书封面：outputs/<name>-xiaohongshu-cover.png
```

除非用户主动询问，不要报告私有参考图细节。
